Fase 2:
"Actúa como un desarrollador backend senior experto en Node.js, NestJS y Prisma. Estamos en la Fase 2 de nuestro proyecto para la prueba técnica (gestor de contactos). Quiero que ejecutes los comandos y generes el código necesario para construir la API REST. Por favor, sigue estos pasos secuencialmente en mi directorio actual:

Paso 1: Inicialización

    Crea un nuevo proyecto NestJS dentro de una carpeta llamada backend. Usa npm como gestor de paquetes. Asegúrate de configurarlo para no inicializar un nuevo repositorio git anidado, ya que estamos en un monorepo.

    Una vez creado, entra a la carpeta backend e instala Prisma como dependencia de desarrollo (npm install -D prisma) y el cliente de Prisma (npm install @prisma/client).

    Instala también las dependencias de validación: npm install class-validator class-transformer.

Paso 2: Configuración de Base de Datos

    Inicializa Prisma (npx prisma init).

    Modifica el archivo .env generado en la carpeta backend para que la variable DATABASE_URL apunte a nuestro contenedor local: DATABASE_URL="postgresql://test:test@localhost:5432/test?schema=public"

    Modifica el archivo prisma/schema.prisma. Asegúrate de que el provider del datasource sea "postgresql". Crea el siguiente modelo:

Fragmento de código

model Contact {
  id        Int      @id @default(autoincrement())
  name      String
  email     String   @unique
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}

    Ejecuta la migración inicial para crear la tabla en la base de datos: npx prisma migrate dev --name init.

Paso 3: Lógica de la API

    Genera un servicio global de Prisma para que NestJS pueda inyectarlo: crea un archivo prisma.service.ts en la carpeta src que extienda PrismaClient e implemente OnModuleInit.

    Genera un recurso RESTful completo para los contactos usando el CLI de Nest: npx @nestjs/cli g resource contacts --no-spec (selecciona REST API y generar operaciones CRUD).

    En src/contacts/dto/create-contact.dto.ts, define las propiedades name y email. Usa los decoradores @IsString(), @IsNotEmpty() para el nombre, y @IsEmail() para el correo importados de class-validator.

    Implementa el ContactsService para que use el PrismaService. Debes crear los métodos para listar todos los contactos (findAll), crear un contacto (create) y eliminar un contacto por ID (remove).

    Actualiza el ContactsController para exponer las rutas correspondientes a esos tres métodos (GET, POST, DELETE).

    En el archivo src/main.ts, habilita CORS (app.enableCors()) para que el frontend pueda consumir la API, y configura el ValidationPipe global (app.useGlobalPipes(new ValidationPipe())) para que los DTOs funcionen.
